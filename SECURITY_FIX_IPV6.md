# Отчет об исправлении уязвимости IPv6 в Chisel

## Проблема
Было обнаружено, что приложение chisel при работе с localhost может неявно использовать IPv6 адрес `::1` вместо IPv4 `127.0.0.1`. Это создавало потенциальную уязвимость безопасности, позволяющую:
- Подбор паролей SSH через IPv6 интерфейс
- Обход некоторых конфигураций безопасности
- Неожиданное поведение при настройке туннелей

## Найденные уязвимые места

### 1. `/workspace/share/settings/remote.go`
**Проблема**: При использовании hostname "localhost" система могла разрешать его как в IPv4, так и в IPv6 адрес.

**Решение**: Добавлена явная конвертация "localhost" в соответствующие IPv4 адреса:
```go
// Предотвращение неявного использования IPv6
// Явно указываем версию IP для избежания подбора паролей через ::1
if r.RemoteHost == "localhost" {
    r.RemoteHost = "127.0.0.1"
}
if r.LocalHost == "localhost" {
    r.LocalHost = "0.0.0.0"
}
```

### 2. `/workspace/share/tunnel/tunnel_out_ssh.go`
**Проблема**: Функция `handleTCP` использовала `net.Dial("tcp", ...)` что позволяло системе выбирать между IPv4 и IPv6.

**Решение**: Изменено на явное использование IPv4:
```go
// Явное указание IPv4 для предотвращения подключения через ::1 (IPv6 localhost)
// Это предотвращает потенциальные атаки подбора паролей SSH через IPv6
dst, err := net.Dial("tcp4", hostPort)
```

### 3. `/workspace/share/tunnel/tunnel_in_proxy.go`
**Проблема**: Функция `listen()` использовала `net.ResolveTCPAddr("tcp", ...)` и `net.ListenTCP("tcp", ...)` что позволяло прослушивание на IPv6.

**Решение**: Изменено на явное использование IPv4:
```go
// Явное использование IPv4 для предотвращения прослушивания на ::1
addr, err := net.ResolveTCPAddr("tcp4", p.remote.LocalHost+":"+p.remote.LocalPort)
...
l, err := net.ListenTCP("tcp4", addr)
```

## Рекомендации

1. **Для пользователей**: После обновления все подключения к "localhost" будут явно использовать IPv4
2. **Для разработчиков**: При добавлении нового сетевого кода используйте явные указания протокола ("tcp4", "udp4")
3. **Мониторинг**: Проверьте логи на наличие попыток подключения через IPv6 после развертывания исправления
